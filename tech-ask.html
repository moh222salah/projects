01
مشكلة حرجة
المواقع التجريبية مكشوفة على Google
لاحظت إن test1 و test مش محمييين بـ noindex ولا بـ Basic Auth — يعني Google بيشوفهم وبيعتبرهم محتوى مكرر مع الموقع الأصلي... ده بيضر الـ SEO بشكل صامت. إيه اللي منعك إنك تعمل حماية لهم من الأول؟
Critical Issue
Staging Sites Exposed to Google
I noticed that test1. and test. subdomains have no noindex meta tag and no HTTP Basic Authentication — meaning Google is actively crawling them as duplicate content. This silently tanks the SEO score. Was there a reason this wasn't addressed, or was it simply overlooked?
02
أداء ضعيف
الموقع بطيء بشكل واضح
لما بفتح الموقع بحس إنه بياخد 4-5 ثواني قبل ما أشوف أي حاجة — ده مش طبيعي خالص. هل اشتغلتِ على تحسين الـ Core Web Vitals في أي مرحلة؟ ولا الموضوع ما اتذكرش أصلاً؟
Performance
Page Load Speed is Critically Slow
When I open the site, it takes 4 to 6 seconds before anything meaningful renders — well outside acceptable thresholds for a B2B business targeting European clients. Was any performance optimization done? Things like LCP improvement, TTI reduction, or TTFB tuning?
03
كود زيادة
CSS و JS زيادة عن اللزوم
لما بشوف الـ Network Tab، بلاقي CSS و JS بيتحملوا وفيهم 60-70% كود مش بيتستخدم في الصفحة خالص — ده علامة إن الـ Page Builder اتستخدم من غير تنظيف. هل اشتغلتِ على Unused CSS Removal ولا سبتيه زي ما هو؟
Code Bloat
Massive Unused CSS & JavaScript
Looking at the Network tab in DevTools, roughly 60–70% of the CSS and JS loaded is completely unused on most pages. Classic symptom of an unoptimized Page Builder. Was any CSS pruning or tree-shaking done, or was that left as-is?
04
صور غير محسّنة
الصور مش متحسنة خالص
الصور على الموقع لسه بـ JPG أو PNG بأحجام كبيرة — مفيش تحويل لـ WebP ولا Lazy Loading واضح. ده لوحده بيأخر الموقع ثانية أو اتنين. اشتغلتِ على تحسين الصور ولا كانت أولوية تانية؟
Images
Images Are Not Optimized
Images across the site are still served in JPG or PNG at large file sizes, with no WebP conversion and no consistent Lazy Loading. On its own, that adds 1–2 seconds to load time. Was image optimization part of the deliverables, or was it deprioritized?
05
أمان
Security Headers غايبة تماماً
لو فتحت الموقع على securityheaders.com هتلاقيه بياخد Grade F أو D — يعني X-Frame-Options و Content-Security-Policy وغيرهم مش موجودين. ده بيسيب الموقع مكشوف لـ Clickjacking. ده ماكانش ضمن المتطلبات ولا ما اتذكرش؟
Security
Security Headers Are Missing
Running the site through securityheaders.com, it likely scores an F or D — headers like X-Frame-Options, Content-Security-Policy, and Referrer-Policy are absent or misconfigured. That leaves the site exposed to Clickjacking and XSS. Was security hardening scoped into the project?
06
Caching
مفيش Caching حقيقي
كل ما حد بيدخل الموقع، الـ Server بيولد الصفحة من أول وجديد من قاعدة البيانات — مفيش Full Page Cache شغال. ده معناه إن لو جه 100 زائر مع بعض، الموقع ممكن يبطأ أو يوقع. اشتغلتِ على Caching Strategy بأي شكل؟
Caching
No Real Caching Strategy in Place
There's no Full-Page Caching layer active — every request hits PHP and regenerates from the database on the fly. Under any meaningful traffic spike, that's a recipe for slowdowns or a full crash. Was WP Rocket, LiteSpeed Cache, or Redis ever implemented or discussed?
07
Plugins
Plugins متراكمة ومتعارضة
في علامات إن في Plugins كتير شغالة مع بعض وبعضها ممكن بيتعارض — ده بيزود الـ Load Time وبيفتح ثغرات أمنية. كام Plugin كانوا شغالين؟ وهل كانوا كلهم ضروريين فعلاً؟
Plugins
Plugin Bloat & Potential Conflicts
There are signs of significant plugin accumulation — causing both performance degradation and security exposure, especially if some haven't been updated. How many active plugins were running? And were they all genuinely necessary, or did some just accumulate over time?
08
SEO تقني
Schema Markup غايب خالص
الموقع مفيهوش Structured Data — يعني Google مش عارف يعمل Rich Snippets للشركة في نتائج البحث. لشركة B2B بتستهدف السوق الأوروبي ده خسارة كبيرة في الظهور. ده كان ضمن المتطلبات الأصلية ولا ما طلبوهوش؟
Technical SEO
Zero Structured Data / Schema Markup
There's no Structured Data anywhere — no Organization schema, no LocalBusiness, no Service markup. For a company targeting the European B2B market, that means zero chance of Rich Snippets in Google, a direct hit to organic visibility. Was Schema Markup ever in scope?
09
Crawling
robots.txt و Sitemap ناقصين
ملف الـ robots.txt مش بيمنع Google من أنه يزحف على الصفحات اللي المفروض تكون خاصة — والـ Sitemap.xml مش محدَّث بانتظام. ده بيخلي الـ Crawl Budget يتبدد على صفحات مش مهمة. اشتغلتِ على الـ Technical SEO Foundation ولا ما دخلتش في الـ scope؟
Crawling
robots.txt and Sitemap.xml Are Weak
The robots.txt isn't blocking Google from crawling pages that should be private — and the sitemap.xml doesn't appear dynamically generated or regularly updated. That means crawl budget is being wasted on low-value pages. Was Technical SEO Foundation work included in scope?
10
Backup
مفيش Backup Strategy واضحة
لو الموقع اتهاجم أو انهار دلوقتي — في backup جاهز ومتعمل بانتظام؟ ولا الموضوع كان متعمل manually من وقت للتاني؟ لأن من غير Automated Backup الموقع ممكن يضيع من غير رجعة.
Backup
No Automated Backup System
If the site went down or got compromised right now — is there a recent automated backup ready to restore from? Or were backups being handled manually on an ad-hoc basis? Without a scheduled off-server backup system, a single incident could mean total data loss.
11
Version Control
مفيش Git ولا Version Control
هل كان في Git Repository شغال طول المشروع — بـ branching صح وcommit history واضح؟ ولا التعديلات كانت بتتعمل مباشرة على الـ Production من غير أي version control؟ ده مهم جداً لأي developer هياخد الموقع بعد كده.
Version Control
No Git Workflow or Version Control
Was there a Git repository maintained throughout the project — with proper branching, commit history, and documented changes? Or were edits being pushed directly to production without version control? This is critical for handover quality and for any future developer taking over the codebase.
12
Core Web Vitals
الـ Core Web Vitals كلها Failing
لما بشغّل الموقع على Google PageSpeed Insights، الـ LCP فوق 4 ثواني، الـ CLS فوق 0.15، والـ INP برا الـ threshold — وده معناه إن Google بيعاقب الموقع في الترتيب بشكل مباشر. هل الـ Core Web Vitals اتقيسوا أو اتشتغل عليهم في أي مرحلة؟
Core Web Vitals
All Three Core Web Vitals Are Failing
Running the site through Google PageSpeed Insights, all three Core Web Vitals appear in the red — LCP above 4 seconds, CLS above 0.15, and INP outside the acceptable range. Since Google uses these as direct ranking signals, this is actively suppressing search visibility. Was CWV ever benchmarked or targeted?

Handover
The Full Technical Handover Question
For a clean technical handover — is there documentation available? Things like: a list of all plugins and their purpose, any custom code written, hosting credentials, DNS records, and any known bugs or unfinished features that were left open?
